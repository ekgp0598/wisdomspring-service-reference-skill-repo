# HM-020 학습화면-영상

## 기본 정보

| 항목 | 내용 |
| --- | --- |
| 공통 화면 ID | HM-020 |
| 모바일 화면 ID | M-HM-020 |
| 웹 화면 ID | W-HM-020 |
| 화면명 | 학습화면-영상 |
| 화면 유형 | 페이지 |

## 화면 개요

사용자가 영상 콘텐츠를 선택해 진입하는 학습 화면이다. 플레이어를 중심으로 콘텐츠 정보, 이전화/다음화 이동, 노트, 추천 콘텐츠를 제공한다. 디바이스별로 노트 노출 방식에는 차이가 있으나, 핵심 학습 구조는 동일하다.

## 상세 설명

### 1. 플레이어

- 콘텐츠 영역 진입 시 영상이 자동 재생된다.
- 테마 상세 페이지로 이동하는 뒤로가기 버튼을 제공한다.
- 앱 브라우저 또는 웹의 이전 페이지 동작인 경우에는 직전 페이지로 이동한다.
- `전체 회차 보기` 아이콘 클릭 시 회차 목록을 노출한다.

| 디바이스 | 노출 화면 | 방식 |
| --- | --- | --- |
| 모바일 | M-HM-B022 | 바텀시트 |
| 웹 | W-HM-P022 | 팝업 |

- 재생시간은 `현재 재생 시간 / 총 콘텐츠 시간` 형식으로 표시한다.

### 2. 콘텐츠 정보

#### 기본 노출

- 서브테마명
- 콘텐츠 제목

#### 추가 노출

- 콘텐츠 소개글
- 태그
- 강사 사진
- 강사명

#### 인터랙션

- 좋아요
  - 클릭 시 카운트가 증가한다.
  - 아이콘은 filled 상태로 변경된다.
- 북마크
  - 아이콘은 filled 상태로 표시된다.
  - 클릭 시 사용자의 북마크에 추가된다.
- 공유
  - 바텀시트 형태로 노출된다.
  - 제공 항목은 `카카오톡`, `Facebook`, `링크 복사`이다.
  - 각 항목은 아이콘과 텍스트 조합으로 표시한다.
  - `링크 복사` 클릭 시 하단 토스트 `링크가 복사되었습니다.`를 노출한다.
- 비멤버십 회원도 좋아요, 북마크, 공유 기능을 사용할 수 있다.
- 비로그인 사용자가 좋아요, 북마크, 공유를 선택하면 로그인 요청을 노출한다.

### 3. 이전화/다음화 보기

- 이전화와 다음화 이동 영역을 제공한다.
- 노출 항목은 아래와 같다.
  - 이전화 또는 다음화 구분
  - 서브테마 유형명(`theme_sub_nm`)
  - 썸네일
  - 콘텐츠 제목
- 콘텐츠 제목은 최대 2줄까지 노출하고, 초과 시 말줄임 처리한다.

#### 노출 정책

- 이전화와 다음화는 현재 재생 중인 콘텐츠가 속한 같은 테마 안에서만 계산한다.
- 회차는 한 테마 안에서 처음부터 끝까지 순환되는 방식으로 연결한다.
- 현재 재생 콘텐츠가 테마의 첫 번째 회차이면, `이전`에는 같은 테마의 마지막 회차를 노출한다.
- 현재 재생 콘텐츠가 테마의 마지막 회차이면, `다음`에는 같은 테마의 첫 번째 회차를 노출한다.
- 서브테마 유형이 달라지더라도 같은 테마 안의 다음 순서 콘텐츠라면 그대로 연결한다.
- 사용자 화면에는 `TALK`, `BOOK`, `ESSAY` 코드값이 아니라 관리자가 입력한 서브테마 유형명(`theme_sub_nm`)을 노출한다.
- 예시:
  - 현재 재생 콘텐츠가 1화일 때 `이전`에는 현재 테마의 마지막 회차가 표시된다.
  - 현재 재생 콘텐츠가 12번째 에세이일 때 `다음`에는 현재 테마의 첫 번째 영상이 표시된다.
- 더 이상 이전 콘텐츠가 없거나 마지막 콘텐츠라는 고정 문구는 노출하지 않는다.

### 4. 노트

- 데스크톱 학습화면 내에 노트를 직접 노출한다.
- 모바일은 노트를 바텀시트로 노출한다.

| 디바이스 | 노출 화면 | 방식 |
| --- | --- | --- |
| 모바일 | M-HM-B023 | 바텀시트 |
| 웹 | W-HM-020 내 영역 노출 | 인라인 |

#### 노트 리스트

- 최신순 기준 상단부터 노출한다.
- 한 번에 최대 10개까지 노출한다.
- 추가 노트는 페이지네이션으로 처리한다.
- 총 노트 개수를 함께 표시한다.
- 각 노트의 구성 항목은 아래와 같다.
  - 프로필 사진
  - 닉네임
  - 노트 내용
  - 좋아요 수
- 노트 내용은 최대 2줄까지 노출하고, 초과 시 말줄임 처리한다.
- 좋아요 아이콘 클릭 시 카운트가 증가한다.
- 좋아요 아이콘은 filled 상태로 변경된다.
- 노트 댓글 아이콘 클릭 시 해당 노트 댓글 팝업을 노출한다.
- 말줄임 여부와 무관하게 노트 본문 또는 댓글 아이콘 선택 시 전체 노트/댓글 팝업을 노출한다.
- PC 노트 리스트에서 본인 노트에는 더보기 메뉴를 제공하고 수정/삭제 진입을 제공한다.
- 본인 노트의 그레이 배경 처리는 사용하지 않는다.

| 디바이스 | 노출 화면 |
| --- | --- |
| 웹 | W-HM-P024 |

#### 노트 입력

- 노트 입력 필드를 제공한다.
- 줄바꿈은 허용하지 않는다.
- 입력 필드 높이는 최대 3줄까지 확장된다.
- `ENTER` 입력 시 노트가 등록된다.
- 하나의 콘텐츠에 여러 노트를 작성할 수 있다.
- 영상/아티클에 질문 데이터 시퀀스가 없어도 노트 입력이 가능해야 한다.
- 콘텐츠 노트 입력은 `goods_id`, `sa_seq` 기반 입력 방식을 지원한다.

#### 노트 없음 상태

- 등록된 노트가 없을 경우 아래 문구를 노출한다.
- `등록된 리뷰가 없습니다. 첫 번째 리뷰를 작성해보세요`

### 5. 추천 콘텐츠

- 현재 재생 중인 콘텐츠와 연관된 로직으로 추천 콘텐츠를 노출한다.
- 추천은 현재 테마와 무관하게 구성될 수 있다.
- 추천 대상은 `TALK`, `BOOK`, `ESSAY`를 모두 포함한다.
- 구성 항목은 아래와 같다.
  - 테마명
  - 서브테마 유형명(`theme_sub_nm`)
  - 썸네일
  - 콘텐츠명

## 비고

- 추천 콘텐츠의 상세 추천 로직은 별도 정의가 필요하다.
