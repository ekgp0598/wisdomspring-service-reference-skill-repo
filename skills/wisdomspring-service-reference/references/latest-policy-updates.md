# Latest Policy Updates

이 문서는 Jira QA 과정에서 확정되었거나 보완된 최신 기획/정책 기준이다. 기존 화면 정의와 충돌하는 경우 이 문서의 내용을 우선 적용한다.

## 홈

- 홈 테마카드는 `테마 오픈일 도래`와 `테마 노출 ON`을 모두 만족할 때만 노출한다.
- 오픈일이 도래하지 않은 테마는 노출 ON 상태여도 홈에 노출하지 않는다.
- 홈 테마카드의 `NEW` 뱃지는 오픈일부터 7일 이내에만 노출한다.
- 오픈 8일차부터 `NEW` 뱃지를 노출하지 않는다.
- `NEW` 뱃지 UI는 API의 `is_new` 값을 기준으로 표시한다.
- 메인 키비주얼 카드는 루프형 인터랙션을 적용한다.
- 첫 카드는 중앙에 위치하고, 왼쪽에는 마지막 카드가 일부 보이는 구조를 사용한다.
- 오늘의 추천 북러닝 카드에는 `BOOK1`, `BOOK2` 같은 유형 코드가 아니라 `서브테마 제목(theme_sub_title)`을 노출한다.
- 내가 시청중인 영상 영역에는 사용자용 서브테마명(`theme_sub_nm`)을 노출한다.

## 인트로와 홈 이동

- 인트로는 앱 재접속 등 최초 진입성 케이스에서만 노출한다.
- 하단 HOME 탭 선택, 홈에서 타 화면 이동 후 뒤로가기 등 일반 홈 진입 케이스에서는 인트로를 노출하지 않는다.

## 테마 상세

- COMING SOON 서브테마와 콘텐츠는 오픈 전 재생할 수 없다.
- COMING SOON 상태에서도 정보 일부 조회는 허용할 수 있다.
- COMING SOON 상태에서는 북마크 버튼을 노출하지 않고, 콘텐츠 재생 및 학습화면 진입을 차단한다.
- COMING SOON 콘텐츠는 전체 회차 보기에서 테마 전체 구성을 보여주기 위해 노출한다.
- 전체 회차에서 COMING SOON 콘텐츠는 `COMING SOON` UI를 표시하고 클릭을 차단한다.
- 오픈 여부는 API의 `open_yn` 기준을 사용한다.
- 현재 재생중 콘텐츠 CTA 문구는 `이어서 보기`를 사용하지 않고 `이어보기`로 통일한다.
- 미시청 상태 CTA는 `처음부터 보기`이다.
- 시청 이력 있음 상태 CTA는 `이어보기`이다.
- 완료 상태 CTA는 `처음부터 다시보기`이다.
- 에세이도 현재 재생중 콘텐츠 영역에 노출한다.
- 미열람 에세이 진행률은 0%로 처리한다.
- 열람 완료 에세이 진행률은 100%로 처리한다.
- 테마의 마지막 콘텐츠 완료 시 `처음부터 다시보기`를 노출하고, 클릭 시 테마 첫 콘텐츠로 이동한다.
- 현재 재생중 콘텐츠 뱃지는 코드값이 아니라 관리자가 입력한 서브테마 유형명(`theme_sub_nm`) 기준으로 표시한다.
- 서브테마/콘텐츠 영역에서는 `TALK`, `BOOK`, `ESSAY` 코드값 대신 관리자가 입력한 서브테마 유형명(`theme_sub_nm`)을 노출한다.
- 햄버거 메뉴의 테마보기 리스트는 홈 테마카드 노출 기준과 동일하다.
- 햄버거 메뉴의 테마 오픈 상태는 `오픈일 도래 + 노출 ON` 기준이며, API의 `open_yn` 값을 사용한다.

## 콘텐츠 학습화면

- 이전화/다음화는 같은 테마 안에서만 계산한다.
- 첫 회차와 마지막 회차는 순환 연결한다.
- 마지막 콘텐츠의 다음은 첫 콘텐츠이고, 첫 콘텐츠의 이전은 마지막 콘텐츠이다.
- 학습 완료 시 기존 완료 팝업은 노출하지 않는다.
- 완료 안내는 토스트 등 가벼운 방식으로 처리한다.
- 추천콘텐츠와 전체보기 등 콘텐츠 카드의 서브테마 노출값은 `theme_sub_nm` 기준이다.
- `theme_sub_type_nm` 코드값을 사용자 화면에 노출하지 않는다.
- 비멤버십 회원도 콘텐츠 좋아요, 저장, 공유 기능을 사용할 수 있다.
- 비로그인 사용자가 좋아요, 저장, 공유를 선택하면 로그인 요청을 노출한다.
- 영상/아티클에 질문 데이터 시퀀스가 없어도 노트 입력이 가능해야 한다.
- 콘텐츠 노트 입력은 `goods_id`, `sa_seq` 기반 입력 방식을 지원한다.

## 전체 회차

- 전체 회차에서는 COMING SOON 콘텐츠 UI를 노출한다.
- 오픈 전 콘텐츠는 표시하되 클릭 및 재생은 불가하다.
- 영상과 에세이를 모두 포괄하는 상태 문구가 필요할 때는 `재생중`보다 `학습 중`을 우선 사용한다.

## 오늘의 질문과 답변

- 모바일에서 답변하기 선택 시 현재 화면 위에 바텀시트를 노출한다.
- 모바일에서 저장 후 AI 피드백보기 팝업 없이 답변조회 상세로 이동한다.
- 모바일 답변조회 상세의 상단 뒤로가기 선택 시 오늘의 질문 페이지로 이동한다.
- PC에서 답변하기 선택 시 팝업을 노출한다.
- PC에서 저장 후 오늘의 질문 페이지로 이동하며 답변조회 팝업을 노출한다.
- PC 답변조회 팝업 닫기 선택 시 팝업만 닫고 오늘의 질문 페이지를 유지한다.
- 홈의 오늘의 질문 배너 영역 클릭 시 오늘의 질문 페이지로 이동한다.
- 홈의 오늘의 질문 배너 내 답변보기 아이콘 클릭 시 내 답변조회를 노출한다.
- 답변 작성 완료 상태에서 `내 답변 보기`는 위치나 리스트 접힘/펼침 상태와 무관하게 항상 답변조회로 진입한다.
- 답변 작성 완료 상태에서 `답변 수정하기` 모달을 노출하지 않는다.
- 답변조회에서 AI 답변보기 영역은 기본 펼침 상태로 노출한다.
- 노트 상세, 오늘의 질문 상세에서 사용자 프로필 선택 시 해당 사용자 프로필로 이동한다.

## 노트와 댓글

- 하나의 콘텐츠에 여러 노트를 작성할 수 있다.
- 기존 `이미 댓글을 작성하였습니다` 방식의 노트 중복 차단은 사용하지 않는다.
- 말줄임 처리된 노트는 텍스트 영역 또는 댓글 아이콘 선택 시 전체 내용/대댓글 팝업으로 진입한다.
- 말줄임 여부와 무관하게 노트 본문 선택 액션은 동일하게 처리한다.
- 본인이 작성한 오늘의 질문 답변, 노트, 각 유형별 댓글은 목록/상세에서 수정 및 삭제할 수 있다.
- 단, 오늘의 질문 답변과 노트는 댓글이 달린 경우 삭제할 수 없다.
- PC 노트 리스트에서 본인 노트에는 더보기 메뉴를 제공하고 수정/삭제 진입을 제공한다.
- 본인 글임을 표시하던 그레이 배경 처리는 사용하지 않는다.
- 본인 여부는 프로필 컬러라인 등 별도 표시 방식으로 구분한다.
- 모바일 노트/댓글 바텀시트 내부 스크롤 시 부모 학습화면이 함께 스크롤되지 않아야 한다.
- 댓글 끝 이후 불필요한 빈 공간 스크롤이 생기지 않아야 한다.

## 프로필과 팔로우

- 팔로우 상태 UI는 `팔로우`와 `팔로잉`을 명확히 구분한다.
- `팔로우` 클릭 시 즉시 `팔로잉` 상태로 반영한다.
- `팔로잉` 클릭 시 팔로우 해제 상태로 반영하고 `팔로우`로 되돌린다.
- 웹 프로필 상단 UI는 내 프로필과 타인 프로필 구성을 분리한다.
- 타인 프로필에서는 팔로잉 여부 정보를 우선 노출한다.

## 멤버십

- 결제 취소/환불 완료 상태는 `해지 완료`가 아니라 미가입 상태로 처리한다.
- 결제 취소/환불 완료 상태에서는 `현재 멤버십이 아닙니다` 문구를 노출한다.
- 결제 취소/환불 완료 상태에서는 값 없는 구독정보 영역을 노출하지 않는다.
- 해지예약 상태를 별도로 제공한다.
- 구독중, 구독중(해지예약), 사용정지, 미가입/결제취소 상태를 분리한다.
- 각 멤버십 상태별 CTA와 구독정보 노출 기준을 분리해 판단한다.

### 취소/환불 규정

- 적용 대상은 위즈덤스프링 멤버십 상품이다.
- 결제 후 7일 이내이고 이용 이력이 없으면 전액 환불한다.
- 결제 후 7일 이내이고 이용 이력이 있으면 결제 금액에서 취소 수수료 10%를 차감 후 환불한다.
- 결제 후 7일 초과이고 이용 이력이 없으면 결제 금액에서 취소 수수료 10%를 차감 후 환불한다.
- 결제 후 7일 초과이고 월간 멤버십 이용 이력이 있으면 환불 불가이다.
- 결제 후 7일 초과이고 연간 멤버십 이용 이력이 있으면 전체 이용기간 중 잔여 이용기간에 해당하는 금액에서 10%를 차감 후 환불한다.
- 잔여 이용기간은 구독 시작일과 종료일을 기준으로 일할 계산한다.
- 결제 후 7일 이내는 플랫폼을 통한 환불 가능 기간이다.
- 결제 후 7일 초과는 플랫폼에서 직접 환불할 수 없다.
- 결제 후 7일 초과 환불 요청은 채널톡 또는 고객행복센터 1588-6559로 접수한다.
- 고객센터 접수 시 가입 계정 확인 후 환불 가능 여부와 환불 금액을 안내한다.

## 공통 로그인

- 비로그인 사용자가 오늘의 질문 작성, 콘텐츠 미리보기 등 권한이 필요한 액션을 선택하면 로그인 요청 팝업을 노출한다.
- 로그인 요청 팝업은 최신 디자인 기준의 여백과 버튼 구성을 따른다.

## 어드민과 데이터 입력

- 서브테마 관리 워딩을 정리한다.
- 프론트 노출용 서브테마 유형명, 서브테마명, 서브테마 제목의 의미를 분리한다.
- 테마/서브테마/콘텐츠/오늘의 질문 등의 입력 글자수 제한과 줄바꿈 허용 여부를 최신 기준으로 적용한다.
- 자동 줄바꿈은 삭제한다.
- 콘텐츠 설명은 최대 200자 기준이다.
- 에세이 요약은 최대 500자 기준이다.
- 테마 소개는 최대 200자 기준이다.
- 서브테마 소개는 최대 110자 기준이다.
